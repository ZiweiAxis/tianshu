# 天枢：投递与消费 — 当前实现状态

**说明**：子项目**运行态**「要做的内容」从哪里拿：以本文及根仓 `子项目任务消费快速参考.md` 为准。规划侧开发待办见根仓 `子项目任务下发与查看约定.md`。

---

## 1. 当前实现

- **投递**：`POST /api/v1/delivery/approval-request`（审批请求投递到 Matrix 房间）、`send_agent_message`（向某 Agent 的 Matrix Room 发消息）。
- **delivery_log**：记录投递开始/完成（delivery_id、semantic_type、target、status）。

---

## 2. 子项目如何接收

**唯一方式**：接入该 Agent 的 **Matrix Room**，收天枢通过 `send_agent_message` 发来的消息；审批类由谛听调 approval-request，天枢投递到 DELIVERY_ROOM_ID，在对应房间内查看。

平台仅通过 Matrix 收消息及既有投递接口。
