# 天枢：与当前方案目标相关的待办任务

**目的**：根据根仓与前面讨论的**方案目标**，列出天枢需要完成的**具体开发/变更任务**，供天枢 Agent 或开发按单执行。设计说明类文档（如 design-alignment、任务与消费-实现状态）仅描述现状与定义，本文给出「需要做什么」。

**依据**：`ziwei/docs/open/technical/谛听在天枢的身份-小谛.md`、`ziwei/docs/open/technical/天枢投递与消费渠道.md`、`ziwei/deploy/危险操作与Matrix审批验证.md`。

---

## 1. 与「危险操作 → Matrix 房间审批」相关的天枢侧任务

| 序号 | 任务 | 说明 | 验收 |
|------|------|------|------|
| T1 | **小谛身份** | 为谛听登记固定身份「小谛」；在接收 `POST /api/v1/delivery/approval-request`（及代谛听发信）时，在投递 payload 或 Matrix 发件人中体现 `sender_id` / `sender_display_name` 为小谛，使用户在房间内看到审批来自「小谛」而非匿名网关。 | 审批消息在 Matrix 房间内展示为来自小谛；实现方式见《谛听在天枢的身份-小谛》§3。 |

说明：天枢已提供 `POST /api/v1/delivery/approval-request` 与 DELIVERY_ROOM_ID，**无需**为「审批进 Matrix 房间」新增接口；谛听侧接入该 API 即可。本任务仅涉及**发件人身份展示**（小谛）。

---

## 2. 与方案目标无关的说明（天枢不需要做）

- **不运维链**：DID 写链/查链由谛听暴露接口，天枢只调用。
- **不做法策决策**：策略与审批等级在谛听；天枢只做触达与回调。
- **不实现监听机制**：监听（鉴权、审批、审计）在谛听；天枢只做通信与身份。
